RelationshipType Class

Defined in: mmrelationships.js:3
Module: gameController

There are just a few instances of this; each one describes a kind of relationship, such as a marriage, an affair, blackmail etc.

Defined in mmrelationships.js:3

Parameters:

  • name String
    • the name of the relationship type, which is also used as a class name in the css
  • directional Boolean
    • if true, the instigator of the relationship does something to the recipient (such as instigator leaves inheritance to the recipient), if false then the relationship is mutual, e.g. a marriage
  • phrase String
    • for displaying a description of the relationship between the instigator and recipient in the form "instigator[phrase]recipient", e.g. phrase could be " is married to "
  • relationshipCalculator Function
    • A function which knows how to assignRandomRelationships and findEnemies for this particular RelationshipType

assignRandomRelationships

(
  • suspectCollection
)
Array

Defined in mmrelationships.js:51

This randomly assigns Relationships of this RelationshipType to the Suspects in the given suspectCollection. In fact this method simply calls the assignRandomRelationships method of the relationshipCalculator, which implements the specific method for this RelationshipType. This means that the caller of this method doesn't need to know what specific RelationshipType it is dealing with (bridge pattern).

Parameters:

  • suspectCollection SuspectCollection

    The singleton holding the collection of Suspects

Returns:

Array:

An array of newly created Relationships

revealRelationshipUnderQuestioning

() Boolean

Defined in mmrelationships.js:76

The user clicks on the name of a Suspect in order to "question" that Suspect. The Suspect might reveal a Relationship (s)he is involved in, and this is method is for determining whehter to do so or not. In fact this method simply calls the revealRelationshipUnderQuestioning method of the relationshipCalculator, which implements the specific method for this RelationshipType. This means that the caller of this method doesn't need to know what specific RelationshipType it is dealing with (bridge pattern).

Returns:

Boolean:

true if a Relationship should be revealed, false if not
